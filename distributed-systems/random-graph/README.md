# 2.2.2.2 非结构化对等体系结构实验

本实验实现《分布式系统原理与范型（第二版）》中文译版第 2.2.2.2 节介绍的非结构化 P2P 成员管理协议，对应教材图 2-9 的主动线程与被动线程。

## 实验目标

非结构化 P2P 系统中的每个节点只维护一个大小为 `c` 的邻居列表，教材称为 partial view。节点并不知道完整的网络成员列表，而是定期选择 view 中的一个节点，双方交换一部分条目，使局部的邻居引用逐渐混合成近似随机的覆盖网络。

1. 每个节点具有独立的 partial view，条目由节点编号和 age 组成。
2. 每轮中，每个存活节点以随机次序发起一次交换。
3. 主动节点从自己的 view 中选择一个 peer。
4. 双方分别构造缓冲区：加入自身地址，再加入约 `c/2` 个 view 条目。
5. 双方以 push-pull 方式交换缓冲区，并各自构造新的 partial view。
6. 新 view 中所有条目的 age 加一。
7. 节点离开时不通知其他节点；其他节点选择到它并发现其不再响应后，才删除该失效引用。

初始化时使用局部环形邻居，而不是直接生成理想随机图。这样可以观察协议如何依靠局部交换打散初始拓扑。

## 书中伪代码

`H` 是 healing 参数，表示每次构造发送缓冲区时移到 partial view 末尾的最旧条目数量。

主动线程的动作（周期性重复）：

```go
从当前 partial view 中选择一个对等节点 P；

if PUSH 模式 {
    mybuffer = [(MyAddress, 0)]；
    随机排列 partial view；
    将 H 个最旧的条目移到末尾；
    将前 c/2 个条目追加到 mybuffer；
    将 mybuffer 发送给 P；
} else {
    向 P 发送触发消息；
}

if PULL 模式 {
    接收 P 的缓冲区；
}

根据当前 partial view 和 P 的缓冲区构造新的 partial view；
将新 partial view 中每个条目的 age 加一；
```

被动线程的动作：

```go
接收来自任意进程 Q 的缓冲区；

if PULL 模式 {
    mybuffer = [(MyAddress, 0)]；
    随机排列 partial view；
    将 H 个最旧的条目移到末尾；
    将前 c/2 个条目追加到 mybuffer；
    将 mybuffer 发送给 Q；
}

根据当前 partial view 和 Q 的缓冲区构造新的 partial view；
将新 partial view 中每个条目的 age 加一；

> 两种请求可能同时到达，所以要保证两种动作是原子串行的

```

## 交换过程

1. 旧引用向后放，先传播交换新引用
2. 删除已经交换的，放入交换后的

新节点优先级高可以快速传播，但可能导致引用聚集在频繁上下线的节点

具体

1. **选择发送条目**：随机排列当前 view，把 `H` 个最旧条目移到末尾，再从前面选择约 `c/2` 个条目放入发送缓冲区。
2. **构造新 view**：合并当前 view 与收到的缓冲区，删除自身引用并对重复引用保留 age 更小的版本。
3. **为新条目腾出空间**：如果合并结果超过容量 `c`，优先删除本次已经发送的条目，使交换表现为双方转移一部分邻居引用。
4. **淘汰旧引用**：如果删除已发送条目后仍然超过容量，再删除 age 最大的条目，直到 view 恢复为 `c` 个条目。

- `age`：引用的新鲜程度，`H`选择时参照的对象。节点每次参与 partial view 更新后，其中所有条目的 age 加一；节点发送自己的地址时，该条目的 age 为 0。

## 运行

环境要求：Go 1.22 或更高版本，无第三方依赖。

```bash
cd distributed-systems/random-graph
go run . \
  -nodes 100 \
  -view 10 \
  -rounds 30 \
  -heal-oldest 1 \
  -fail-round 15 \
  -failures 10 \
  -report-every 5 \
  -seed 20260718
```

主要参数：

- `-nodes`：初始节点总数。
- `-view`：partial view 的容量 `c`。
- `-rounds`：协议运行轮数。
- `-heal-oldest`：设置 healing 参数 `H`；取值范围为 `0` 到 `c-1`，默认值为 `1`。
- `-fail-round`：令节点离开的轮次；设为 `0` 表示不注入离开事件。
- `-failures`：离开的节点数。
- `-report-every`：每隔多少轮输出一次统计。
- `-seed`：随机种子，用于复现实验。

## 输出指标

```text
round  event     alive  component   indeg    stddev range       age deadrefs
    0  initial     100    100/100     10.00    0.00   10..10      0.00       0
```

- `alive`：当前存活节点数。
- `component`：从首个存活节点所在的弱连通分量大小。只要一方的 view 中保存另一方，双方就能建立通信，因此这里按无向连接检查覆盖是否分裂。
- `indeg`：存活节点平均入度，即平均有多少其他 partial view 引用该节点。
- `stddev` 和 `range`：入度标准差及最小值、最大值，用来观察引用是否过度集中到少数热门节点。
- `age`：存活节点所维护条目的平均年龄。
- `deadrefs`：存活节点 view 中仍然存在的失效节点引用数。

在 `fail-round` 后，重点观察 `deadrefs` 是否逐轮下降，以及 `component` 是否仍等于 `alive`。这可以同时验证邻居交换、旧引用淘汰和覆盖连通性。

## 标准随机图

标准随机图通常指 Erdős-Rényi 随机图模型，常见形式为 `G(N, p)`：

- `N` 表示图中的顶点总数。
- 任意两个不同顶点之间都独立地以概率 `p` 建立一条边。
- 每个顶点的度数不是固定值，而是随机变量，其期望为 `p(N-1)`。

例如，在 `G(100, 0.05)` 中，任意两个节点之间存在边的概率是 `0.05`，每个节点的期望度数为 `4.95`。实际度数可能高于或低于这个值。

本实验中的随机覆盖图与标准 Erdős-Rényi 随机图不同：

- 每个节点维护容量为 `c` 的 partial view，因此理想状态下每个节点固定拥有 `c` 条出边。
- 随机的是 partial view 中保存哪些节点，而不是是否创建每一条可能的边。
- 节点 `P` 的 view 包含节点 `Q`，只表示存在有向引用 `P -> Q`，并不自动表示 `Q -> P`。
- 各节点无法直接从完整成员集合中生成理想随机 view，而是通过周期性交换条目，在统计意义上逐渐接近随机覆盖图。

如果只看某个节点 `P` 是否选择一个指定节点 `Q`，在理想的均匀随机 partial view 中，其概率为：

```text
Pr(P chooses Q) = c / (N - 1)
```

因此，可以令 `p = c/(N-1)` 来比较两种模型的平均度数，但二者仍不是同一个随机图模型：`G(N,p)` 的边数和顶点度数都是随机的，而本实验模型固定每个节点的 partial view 容量，只通过协议不断改变其中的节点引用。

> 为什么不是选择最新的一半，而是只让一部分最老不发送，其他机会等价

因为该协议有两个目标：

1. 淘汰可能失效的旧引用。
2. 保持邻居选择近似均匀随机。

如果直接选择“最新的一半”，age 会成为主要选择条件，导致新引用被反复传播，旧但仍有效的节点几乎永远没有传播机会。概率和引用快速收敛到最新的节点上。

> 新引用会逐渐变成旧引用，成为最旧后不被其他节点传播。会不会没传全图，就死掉了

会，某一条具体引用可能还没传播多远，就因 age 过大被淘汰。但协议并不要求每条引用传遍全图。

每个节点只维护 c 个邻居，因此某个节点的地址通常只存在于少量 view 中。理想情况下，一个节点的平均入度约为 c，而不是 N-1。

关键是，存活节点会不断重新产生自己的新引用。节点 Q 每次参与交换时都会发送：

(Q.address, age=0)

这个自身引用不受 H 限制。因此过程是：

Q 的旧引用逐渐老化并消失
            同时
Q 不断产生 age=0 的新引用并传播

即使某一时刻没有任何其他节点引用 Q，只要 Q 自己还保存着可联系的活邻居，它仍能主动发起交换，把自己的地址重新加入对方的 view。

真正危险的是节点的 view 中已经没有任何可用活邻居，或者覆盖网络发生了分区。

所以，单条引用可以“死掉”，但存活节点会持续生成新的自身引用。**协议依靠许多引用副本的产生和淘汰达到动态平衡**，而不是让一条引用负责传播到全图。


老化标记大小和速度不影响节点边的概率，因为都扩大=没扩大，只影响边的传播深度和覆盖面。如果大，意味着更大可能频繁的向新边建立和传播，而不是沿着维持已有边传播。
